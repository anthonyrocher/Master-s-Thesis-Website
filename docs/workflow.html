Workflow

This page describes the data management of this work, from the pre-processing steps to the results. The goal is to write step by step a replication package. However, note that some data are not public and should be required to their owners (namely the LIFFULL HOMES dataset the panel dataset from Keio University).

Code used in this section: Python, SQL and Stata.

Let’s clean the Liffull Homes datasets

The Liffull Homes data consists of actually two types of datasets. The first one, that I will call “monthly”, includes all the listings from July 2015 to June 2017. The second one, “period”, includes all the listings from October 2018 to December 2014. Unfortunately, those two do not match perfectly well.

You might notice that for years 2015, 2023 and 2024, the associated shapefiles covers some other years. The reason is that they do not exist. I decide to take the previous risk map for 2014 and the 2022 for 2013 and 2024, acknowledging that this is a lesser evil in my research design, but arguing that this should not be harmful since those maps, even though evolving year by year, are computed using a Poisson process over decades and do not move that much from one year to another.

The important thing to pay attention to is to watch out for CRS issue. The coodebook says that the data are EPSG:4326 encoded. However, the maps are EPSG:4612. Those two differ only by a few meters but still let’s convert during merging.

    with open(file_input, "r", encoding = "utf-8") as f:

Second, I use the documentation provided by the JMA (https://www.data.jma.go.jp/eqev/data/bulletin/data/shindo/format_e.txt) to parse the lines.

def parse_line(line):
    try:
        return {
            "date":             f"{line[1:5]}-{line[5:7]}-{line[7:9]}",
            "magnitude":        line[52:54].strip(),
            "max_intensity":    line[61].strip(),
            # The CRS is once again different. The following formula converts from DMS to WGS84 (epsg:4326) format.
            "eq_lat":           int(line[21:24]) + (int(line[24:28]) / 100) / 60,
            "eq_lon":           int(line[32:36]) + (int(line[36:40]) / 100) / 60,
            "damage_class":     line[62].strip(),
            "tsunami_class":    line[63].strip(),
            "district_number":  line[64].strip(),
            "region_number":    line[65:68].strip(),
        }
    except:
        return None

with open("/Users/anthony/Documents/master_thesis/data/raw/jma/jma_combined.txt", "r", encoding="utf-8") as f:
    lines = f.readlines()

df = pd.DataFrame(data)

Many values are missing. For example, the variables from the “population” category come from the Japanese census which happens every five years. I proceed three ways to fill in missing variables:

linear interpolation
for data available only for years 2015 and 2020, mainly the ones from the census, I compute the linear line coefficients between those two variables and complete the missing variables depending on the year. 
********************************************************************************
* FILLING IN MISSING VALUES

* Linear interpolation
* Those for which 2015 and 2020 are available
foreach var of varlist pop_total median_age lifeexp_male lifeexp_female pop_u15 pop_o65 did_pop foreigners households pop_total2 employed unemployed labor_force skilled_workers selfemp_with selfemp_without commute_in commute_out {

    * Create base
    gen `var'_base = .
    bysort areacode (year): replace `var'_base = `var'[1]

    * Linear interpolation coefficients
    gen `var'_delta = .
    bysort areacode (year): replace `var'_delta = (`var'[6] / `var'[1]) ^ (1/5)

    * Interpolation computation
    gen `var'_linear = .
    forvalues y = 2015/2023 {
        local t = `y' - 2015
        replace `var'_linear = `var'_base * (`var'_delta ^ `t') if year == `y'
    }

    * Drop temporary var
    drop `var'_delta
    drop `var'_base
}
Regression
this method is based on the previous one, but more than two years are available. 
* Regression
gen inmigrants_reg = inmigrants
gen outmigrants_reg = outmigrants
foreach var in inmigrants outmigrants {

    gen yhat = .

    levelsof areacode, local(areas)

    foreach a of local areas {
        count if areacode == `a' & !missing(year, `var')
        
        if r(N) >= 2 {
            regress `var' year if areacode == `a'
            predict temp if areacode == `a'
            
            replace yhat = round(temp) if areacode == `a'
            drop temp
        }
        else {
            display "Skipping areacode `a' for `var' (not enough data)"

Nearest value
For the rest of the variables, if a observation is missing, I take the value of the previous year. For example, if some city had 2 libraries in 2015 and 2016 is N.A, then I suppose I also had 2 libraries in 2016. 
* Nearest values

    bysort areacode (year): replace `var'_near = `var'_near[_n+1] if missing(`var'_near[1])
    bysort areacode (year): replace `var'_near = `var'_near[_n-1] if missing(`var'_near) 
}
